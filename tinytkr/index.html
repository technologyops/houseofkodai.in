tinyTkr

all about you and me ;-)

  1. who am i...
    born in the house of kodai; tinyTkr is a small windoz utility that periodically polls network devices and displays critical color coded values in a ticker tape format. I am small, simple and easy to use; so try me...
     
  2. where am i...
    tinyTkr 0.74e can be downloaded here

  3. use me...
    • Download and extract two files tinytkr.exe and tinytkr.conf
    • populate the configuration file with information about the devices you wish to monitor and
    • execute the tinytkr.exe application
    • Removing/uninstalling is even simpler, just delete both the files
     
  4. your words are my command...
    The configuration file is a plain text file where you define devices and their attributes. The best way to understand the configuration file is to make changes to the sample file and execute the program ;-)

    for those who are technically curious...
    • Configuration files have 4 types of lines: blank lines, comment lines, device lines, and attribute lines
    • Lines that are empty are blank lines
    • Lines that start with the # character are comment lines
    • Device and Attribute lines have values seperated by the space character
      values that have the space character are quoted within a pair of " characters
    • Devices are defined by one device line followed by zero or more attribute lines and are seperated by one or more blank lines
    • tinytkr currently supports 3 types of devices (protocols): snmp, cgi, and tcp
      • snmp: read from standard snmp agents
      • tcp: checks the connection times of tcp based services listed as attributes
      • cgi: used to check the status of parameters (such as free disc space, network connections, memory, etc.) expressed as plain text via cgi
        the network address must be a URL (hostname/resource-path)
        requires the URL to return plain text data as lines of attribute-name value pairs
